# 008-功能按钮沉浸式采集

## 已完成内容
- 将录制入口从常驻底部控制条收纳到“功能”按钮菜单中。
- USB 采集预览开始后自动隐藏控制层，进入全屏沉浸式采集画面。
- 接入返回键/滑动返回手势逻辑：
  - 预览沉浸时返回：显示功能按钮。
  - 功能按钮已显示时返回：再次隐藏控件，回到沉浸画面。
  - 设置面板或菜单展开时返回：优先关闭面板/菜单。
- 功能按钮菜单已包含：
  - 录制/停止录制。
  - 设置采集分辨率/帧速率。
  - 退出采集画面。
- 采集设置面板已包含：
  - 设备、预览、方向、状态。
  - 采集卡/库返回的分辨率列表。
  - `24fps / 30fps / 60fps` 帧速率预设，并注明当前库未暴露采集卡真实 FPS 枚举。
  - 主题切换保留在设置面板内。
- “退出采集画面”会先尝试停止录制，再关闭当前 Activity。
- 已执行构建：`.\gradlew.bat :app:assembleDebug`，构建成功。
- 最新 APK：`D:\Myproject\usb摄像头\app\build\outputs\apk\debug\app-debug.apk`
- 最新 APK 大小：`26213608` bytes。
- 项目根目录 `.gradle` 临时缓存已清理；保留 `app/build`，因为其中包含最新 APK。

### 3. 返回键/滑动返回逻辑

修改前：
```kotlin
// 未单独接管返回键，系统返回按默认 Activity 行为处理。
```

修改后：
```kotlin
onBackPressedDispatcher.addCallback(this, createBackCallback())

private fun handleCaptureBack() {
    if (showFunctionMenu) {
        showFunctionMenu = false
        showControls = true
        return
    }
    if (showSettings) {
        showSettings = false
        showControls = true
        return
    }
    if (uiState.isPreviewing) {
        if (showControls) {
            hideControlsForPreview()
        } else {
            revealControls()
        }
        return
    }
    finish()
}
```
