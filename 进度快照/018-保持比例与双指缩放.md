## 具体修改的代码前后对比

### 1. 竖屏从拉伸铺满改为等比居中

修改前只有 GLES 尺寸更新：

```kotlin
client.setRenderSize(view.width, view.height)
```

修改后增加基础显示矩阵：

```kotlin
val sourceAspect = request.previewWidth.toFloat() / request.previewHeight
val viewAspect = viewWidth.toFloat() / viewHeight
val baseScaleX: Float
val baseScaleY: Float
if (sourceAspect > viewAspect) {
    baseScaleX = 1f
    baseScaleY = viewAspect / sourceAspect
} else {
    baseScaleX = sourceAspect / viewAspect
    baseScaleY = 1f
}
```

16:9 源在 9:16 竖屏中的 `baseScaleY` 约为 `0.316`，最终显示为横向 16:9 画面，上下区域显示黑色背景。

### 2. 从单击层扩展为单击加双指手势

修改前：

```kotlin
Modifier
    .fillMaxSize()
    .clickable(onClick = onPreviewTap)
```

修改后：

```kotlin
Modifier
    .fillMaxSize()
    .previewTransformGestures(
        enabled = !screenLocked,
        onTransform = onPreviewTransform
    )
    .clickable(onClick = onPreviewTap)
```

`previewTransformGestures` 仅在检测到两根按下的指针时计算缩放、平移和中心点并消费事件。

### 3. 缩放和平移边界

```kotlin
val contentWidth = viewWidth * scaleX
val contentHeight = viewHeight * scaleY
val maxPanX = maxOf(0f, (contentWidth - viewWidth) / 2f)
val maxPanY = maxOf(0f, (contentHeight - viewHeight) / 2f)
previewPanX = previewPanX.coerceIn(-maxPanX, maxPanX)
previewPanY = previewPanY.coerceIn(-maxPanY, maxPanY)
```

## 验收标准

- 横屏转竖屏后，16:9 采集画面不变形，居中显示且上下为黑边。
- 竖屏转横屏后，画面仍保持原比例并尽量利用屏幕空间。
- 双指张合可在 `1x–5x` 范围连续缩放。
- 放大后双指拖动可查看左右或上下细节，不能把画面完全拖出屏幕。
- 缩回 `1x` 后画面自动居中。
- 单指点击仍能唤出控制层；锁屏后双指操作不改变画面。
- 录制时缩放只影响本地预览，不改变录制文件分辨率或中断录制。
- USB 重连和切换分辨率后恢复 `1x` 居中。
