- 锁屏状态继续禁用所有预览变换手势。
- 使用 `rememberUpdatedState` 读取实时放大状态，避免 Compose 因状态变化重启手势协程。

## 具体修改的代码前后对比

### 1. 放大状态从渲染层同步到手势层

修改前，缩放倍数只是 Activity 普通字段，Compose 无法判断何时启用单指拖动：

```kotlin
private var previewZoom = MIN_PREVIEW_ZOOM
```

修改后增加仅用于启停手势的 Compose 状态，并在恢复 `1x` 时清零：

```kotlin
private var previewZoom = MIN_PREVIEW_ZOOM
private var isPreviewZoomed by mutableStateOf(false)

previewZoom = newZoom
isPreviewZoomed = newZoom > MIN_PREVIEW_ZOOM
if (!isPreviewZoomed) {
    previewPanX = 0f
    previewPanY = 0f
}
```

### 2. 接近原始比例时吸附回 1x

```kotlin
val scaledZoom = (oldZoom * zoomChange).coerceIn(MIN_PREVIEW_ZOOM, MAX_PREVIEW_ZOOM)
val newZoom = if (scaledZoom <= PREVIEW_ZOOM_SNAP_THRESHOLD) {
    MIN_PREVIEW_ZOOM
} else {
    scaledZoom
}
```

阈值为 `1.03x`，避免手势结束时残留肉眼难以察觉的小倍率，导致单指拖动一直处于启用状态。

### 3. 单指轻点和拖动分流

修改前只处理两根手指：

```kotlin
if (event.changes.count { it.pressed } >= 2) {
    onTransform(zoomChange, panChange.x, panChange.y, centroid.x, centroid.y)
}
```

修改后在放大状态下累计单指位移，超过系统触摸阈值才消费事件并平移；双指手势结束后则允许剩余单指立即接管：

```kotlin
pressedChanges.size == 1 && singleFingerPanEnabled() -> {
    if (hadMultiTouch) singleFingerPanActive = true
    // 未经历双指操作时，累计位移达到 touchSlop 才开始拖动。
    if (singleFingerPanActive) {
        onTransform(1f, panX, panY, change.position.x, change.position.y)
        change.consume()
    }
}
```

## 验收标准

- 预览为 `1x` 时，单指拖动不改变画面位置。
- 放大到大于 `1x` 后，单指可上下左右移动并查看细节。
- 双指缩放后抬起一指，剩余单指可连续移动，不需要重新触摸。
- 缩放到 `1.03x` 以下时自动回到 `1x` 居中，随后单指移动失效。
- 放大状态下轻点仍可正常唤出控制层，不因轻微手抖误触拖动。
- 平移继续受原有画面边界限制，不能将采集内容完全拖出可视区域。
- 锁屏后单指和双指操作均不改变画面。
