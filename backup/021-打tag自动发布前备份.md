# 021-打tag自动发布前备份

## 备份信息

- 日期：2026-07-15
- 阶段：GitHub tag 自动发布 APK 到 Releases
- 目标文件：
  - `.github/workflows/android-release-apk.yml`
  - `README.md`

## 修改前状态

- 仓库已存在 `main` 分支自动构建工作流：`.github/workflows/android-debug-apk.yml`。
- 当前工作流只会在 `push main` 或手动触发时构建 Debug APK，并上传到 Actions Artifacts。
- 当前工作流权限为：

```yaml
permissions:
  contents: read
```

- 这意味着当前 CI 无法创建或更新 GitHub Releases。
- 当前 Releases 页面仍只有手动发布的 `v1.4.0`。

## 当前行为

- 日常 `push main`：
  - 自动构建 APK
  - 上传 Artifact
  - 不创建 Release

- 当前不存在以下能力：
  - `push v* tag` 自动创建 Release
  - 自动把 APK 上传到 Release 资产

## 计划中的最小改动

- 新增独立的 tag 触发工作流，不改现有 `main` 构建工作流的职责。
- 触发条件限定为：
  - `push` 标签 `v*`
  - `workflow_dispatch`
- 工作流只做以下动作：
  - 检出代码
  - 配置 JDK 17、Android SDK、Gradle
  - 执行 `:app:assembleDebug`
  - 复制并重命名 APK 为稳定英文文件名
  - 使用 `gh release create` 或 `gh release upload --clobber` 发布到 GitHub Releases

## 本阶段边界

- 不接入 release keystore。
- 继续发布当前可构建的 Debug APK，而不是 release 签名包。
- 不修改应用业务逻辑。
- 不删除现有 Artifact 上传流程。
