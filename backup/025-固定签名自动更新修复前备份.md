# 025-固定签名自动更新修复前备份

## 本阶段目标

- 修复 GitHub Release 自动更新时出现的“与已安装应用签名不同”问题。
- 为 CI 接入固定签名证书，避免每次 GitHub Actions 运行生成不同的调试签名。
- 发布一个新的稳定签名版本，作为后续自动更新的基线版本。

## 修改前现状

### 当前问题

- 客户端已能检测到更新并下载 APK。
- 系统安装器在升级 `1.4.2 -> 1.4.3` 时提示：
  - `安装失败(-7)`
  - `与已安装应用签名不同`

### 当前 Release 工作流

文件：`.github/workflows/android-release-apk.yml`

- 当前执行的是：

```bash
./gradlew --no-daemon :app:assembleDebug
```

- 当前发布的是 Debug APK。
- GitHub Actions runner 是临时环境，每次构建的默认 debug keystore 都可能不同。

### 当前项目签名配置

文件：`app/build.gradle.kts`

- 目前没有固定的 release signing 配置。
- 因此 CI 产物无法保证和上一版 APK 使用同一证书。

### 本机已有固定 keystore

- 路径：`C:\Users\Administrator\.android\debug.keystore`
- 可作为短期稳定签名基线接入 CI。
