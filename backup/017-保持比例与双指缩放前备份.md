## 修改前问题

### 1. 竖屏画面被拉伸

AUSBC 的 `AspectRatioTextureView.setAspectRatio(width, height)` 在竖屏时会把 16:9 反转成 9:16，导致横向采集画面铺满竖屏并被压窄。

```kotlin
client.getCameraRequest()?.let { request ->
    cameraView.setAspectRatio(request.previewWidth, request.previewHeight)
}
```

### 2. 预览层只支持单击

```kotlin
Box(
    modifier = Modifier
        .fillMaxSize()
        .clickable(
            interactionSource = tapInteractionSource,
            indication = null,
            onClick = onPreviewTap
        )
)
```

### 3. 没有用户缩放状态

当前只向 `RenderManager` 更新宽高，没有预览缩放倍数、平移位置或边界约束。

## 预期修改

- 使用 `TextureView.setTransform(Matrix)` 抵消竖屏拉伸，按采集源比例居中显示。
- 根视图使用纯黑背景，空余区域显示为黑边。
- 支持双指 `1x–5x` 缩放及双指平移。
- 单指点击仍只负责显示控制层。
- 横竖屏、分辨率切换和 USB 重连后重新计算显示矩阵。

## 修改后关键代码

### 1. 保持采集源比例

```kotlin
val sourceAspect = request.previewWidth.toFloat() / request.previewHeight
val viewAspect = viewWidth.toFloat() / viewHeight
if (sourceAspect > viewAspect) {
    baseScaleX = 1f
    baseScaleY = viewAspect / sourceAspect
} else {
    baseScaleX = sourceAspect / viewAspect
    baseScaleY = 1f
}
```

### 2. 双指缩放与焦点平移

```kotlin
val newZoom = (oldZoom * zoomChange).coerceIn(MIN_PREVIEW_ZOOM, MAX_PREVIEW_ZOOM)
val appliedZoomChange = newZoom / oldZoom
previewPanX = previewPanX * appliedZoomChange +
    (1f - appliedZoomChange) * (localCentroidX - centerX) + panChangeX
previewPanY = previewPanY * appliedZoomChange +
    (1f - appliedZoomChange) * (localCentroidY - centerY) + panChangeY
```

### 3. 仅消费双指事件

```kotlin
if (event.changes.count { it.pressed } >= 2) {
    val zoomChange = event.calculateZoom()
    val panChange = event.calculatePan()
    val centroid = event.calculateCentroid()
    onTransform(zoomChange, panChange.x, panChange.y, centroid.x, centroid.y)
    event.changes.forEach { it.consume() }
}
```
