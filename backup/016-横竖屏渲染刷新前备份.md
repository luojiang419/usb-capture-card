## 根因

- `AndroidManifest.xml` 声明了 `orientation|screenSize` 的 `configChanges`，旋转时 Activity 不会重建，USB/UVC 会话能够保留。
- AUSBC 3.2.7 的 `AspectRatioTextureView.setAspectRatio(width, height)` 会读取当前方向：横屏使用 `width / height`，竖屏使用 `height / width`，并调用 `requestLayout()`。
- 重新插拔会经过 `CameraClient.openCamera()`，因此会重新调用 `setAspectRatio()`。
- 当前旋转路径只调用 `CameraClient.setRenderSize()`，既没有重算宽高比，也没有等待新布局完成，所以显示 Surface 仍使用旧方向。

## 修改前代码

```kotlin
override fun onConfigurationChanged(newConfig: Configuration) {
    super.onConfigurationChanged(newConfig)
    screenOrientation = ScreenOrientation.from(newConfig.orientation)
    refreshRenderSize()
}

private fun refreshRenderSize() {
    if (!::cameraView.isInitialized) return
    cameraView.post {
        if (cameraView.width > 0 && cameraView.height > 0) {
            cameraClient?.setRenderSize(cameraView.width, cameraView.height)
        }
    }
}
```

## 预期修改

- 旋转后按当前 `CameraRequest` 重新调用 `cameraView.setAspectRatio()`。
- 使用布局完成回调获取新 `TextureView` 宽高，再调用 `setRenderSize()`。
- 不关闭相机、不重新申请 USB 权限、不停止录制。

## 修改后代码

```kotlin
private fun refreshRenderSize() {
    if (!::cameraView.isInitialized) return
    val client = cameraClient ?: return
    val oldWidth = cameraView.width
    val oldHeight = cameraView.height
    client.getCameraRequest()?.let { request ->
        cameraView.setAspectRatio(request.previewWidth, request.previewHeight)
    }
    cameraContainer.requestLayout()
    cameraView.doOnLayout { view ->
        if (view.width > 0 && view.height > 0) {
            client.setRenderSize(view.width, view.height)
        }
    }
}
```
