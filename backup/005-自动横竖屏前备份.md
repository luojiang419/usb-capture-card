# 005-自动横竖屏前备份

## 阶段
自动横竖屏适配开始前。

## 当前状态
- Manifest 已配置 `android:screenOrientation="fullSensor"`，系统层面允许根据传感器自动横竖屏。
- Activity 使用 `configChanges="keyboardHidden|orientation|screenSize|uiMode"`，横竖屏变化时不会重建 Activity。
- 当前 Compose 控制层没有显式记录方向，横屏和竖屏使用同一套纵向布局。

## 本阶段计划修改
- 在 `MainActivity` 中记录当前屏幕方向。
- 覆写 `onConfigurationChanged()`，横竖屏变化时同步 UI 状态并刷新 AUSBC 渲染尺寸。
- Compose UI 根据方向自适应：竖屏保持当前上下布局，横屏改为更适合宽屏的侧边控制布局。
- 设置面板展示当前方向，方便真机验证。

## 代码前后对比

### 修改前
```kotlin
UsbCaptureScreen(
    uiState = uiState,
    ...
)
```

### 修改后预期
```kotlin
UsbCaptureScreen(
    uiState = uiState,
    screenOrientation = screenOrientation,
    ...
)

override fun onConfigurationChanged(newConfig: Configuration) {
    super.onConfigurationChanged(newConfig)
    screenOrientation = ScreenOrientation.from(newConfig.orientation)
    refreshRenderSize()
}
```
