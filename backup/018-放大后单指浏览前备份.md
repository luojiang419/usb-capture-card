## 修改前行为

- 预览只在检测到两根按下的手指时计算缩放和平移。
- 放大后抬起一根手指，剩余单指不能继续拖动画面。
- `previewZoom` 是普通字段，Compose 手势层不知道当前是否已放大。

```kotlin
if (event.changes.count { it.pressed } >= 2) {
    val zoomChange = event.calculateZoom()
    val panChange = event.calculatePan()
    val centroid = event.calculateCentroid()
    onTransform(zoomChange, panChange.x, panChange.y, centroid.x, centroid.y)
    event.changes.forEach { it.consume() }
}
```

## 预期修改

- `previewZoom > 1x` 时启用单指拖动。
- 使用系统 `touchSlop` 区分单指轻点与拖动，避免破坏点击菜单。
- 双指缩放后可抬起一指并连续拖动。
- 接近 `1x` 时自动吸附到 `1x`、清零平移并禁用单指移动。

## 修改后行为

- 新增 `isPreviewZoomed` 低频状态，只在跨越 `1x` 边界时通知 Compose 是否允许单指平移。
- 缩放结果小于等于 `1.03x` 时吸附到精确 `1x`，同时清零平移并禁用单指拖动。
- 放大状态下，单指移动超过系统 `touchSlop` 后开始平移，轻点仍由原点击逻辑处理。
- 双指缩放后抬起一根手指，剩余手指立即接管平移，不需要重新落指。
- 使用 `rememberUpdatedState` 动态读取放大状态，避免缩放过程中重启 `pointerInput` 并中断当前手势。

```kotlin
pressedChanges.size == 1 && singleFingerPanEnabled() -> {
    val change = pressedChanges.first()
    val deltaX = change.position.x - change.previousPosition.x
    val deltaY = change.position.y - change.previousPosition.y
    if (hadMultiTouch) singleFingerPanActive = true

    if (singleFingerPanActive && (deltaX != 0f || deltaY != 0f)) {
        onTransform(1f, deltaX, deltaY, change.position.x, change.position.y)
        change.consume()
    }
}
```
