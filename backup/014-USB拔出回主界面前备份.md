# 014-USB拔出回主界面前备份

## 备份时间
2026-07-03 优化 USB 采集卡拔出后的界面回退逻辑前。

## 当前阶段
模块十四：USB 拔出后直接回到主等待界面。

## 修改目标
- 拔掉 USB 采集卡后，直接关闭采集相关浮层并回到软件主界面。
- 不再保留设置面板、功能菜单、锁屏状态或沉浸式隐藏系统栏。
- 保持现有单 Activity + Compose 覆盖层结构，不引入新页面或新架构。

## 修改前状态
- USB 断开回调会停止录制计时和 FPS 统计，并把 `isPreviewing`、`isRecording` 置为 `false`。
- 断开回调调用的是 `revealControls()`，该函数只显示控制层，不会统一关闭设置面板、功能菜单或解除锁屏。
- 如果拔出时处于设置面板、功能菜单、锁屏或沉浸状态，用户可能仍需多次滑动/返回才能回到主等待界面。

## 修改前关键代码

### 1. 断开回调

```kotlin
override fun onDetachDec(device: UsbDevice?) {
    runOnUiThread {
        stopRecordingTicker()
        stopPreviewFps()
        revealControls()
        uiState = uiState.copy(
            connectionState = DeviceConnectionState.Waiting,
            statusText = "采集卡已拔出",
            deviceName = deviceLabel(device),
            isPreviewing = false,
            isRecording = false
        )
    }
}

override fun onDisConnectDec(device: UsbDevice?, ctrlBlock: USBMonitor.UsbControlBlock?) {
    runOnUiThread {
        stopRecordingTicker()
        stopPreviewFps()
        revealControls()
        uiState = uiState.copy(
            connectionState = DeviceConnectionState.Waiting,
            statusText = "采集卡连接已断开",
            deviceName = deviceLabel(device),
            isPreviewing = false,
            isRecording = false
        )
    }
}
```

### 2. 控制层显示逻辑

```kotlin
private fun revealControls() {
    if (isScreenLocked) return
    showControls = true
    markControlsInteraction()
}
```
